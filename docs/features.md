# 功能说明

本文档说明 AI 学习助手当前面向用户提供的主要功能，以及各功能在产品中的使用方式。

## 产品定位

AI 学习助手面向需要持续提问、整理学习资料和保存问答记录的用户。系统通过会话组织学习主题，通过流式回复降低等待感，并使用 Markdown 和代码高亮改善技术内容的阅读体验。

## 账号与登录

### 登录

用户可以使用用户名或邮箱登录。登录成功后，前端会保存 JWT 和用户信息，用于保持当前会话状态。

### 注册

新用户可以输入用户名、可选邮箱和密码注册。注册成功后会自动登录并进入聊天页。

### 演示账号

后端启动时会自动创建演示账号：

```text
账号：admin
密码：123456
```

登录页默认填入该账号，便于快速体验项目。

### 注销

用户可以退出当前账号。注销后，本地保存的登录令牌和用户信息会被清除。

## 用户资料

### 默认头像

系统支持默认头像展示。用户未设置头像时，前端可以根据账号信息展示默认头像状态。

### 头像更新

用户可以通过两种方式更新头像：

- 上传本地头像文件
- 提交头像地址

头像更新成功后，系统会返回新的用户信息和登录令牌，前端同步刷新当前用户状态。

### 密码修改

后端提供密码修改接口，要求用户提交旧密码、新密码和确认密码。旧密码校验通过后才会更新密码。

## AI 对话

### 多轮对话

用户可以在同一个会话中连续提问。后端会根据会话记录组织上下文，并将用户问题转发给当前配置的 AI 供应商。

### 流式回复

AI 回复通过流式响应返回。前端会边接收边展示内容，让用户尽早看到生成结果。

### 停止生成

生成过程中用户可以停止当前回复。前端会中断本次请求，并将当前已经生成的内容保留在页面中。

### 重新生成

当当前会话中存在用户消息时，用户可以重新生成上一条 AI 回复。系统会移除最近一条助手回复，并基于最近上下文重新请求 AI。

### Markdown 与代码展示

AI 回复支持 Markdown 渲染，适合展示标题、列表、引用、表格和代码块。代码块支持高亮和复制，便于学习编程类内容。

## 会话管理

### 会话列表

聊天页左侧展示当前用户的会话列表。用户进入聊天页时，系统会加载会话列表，并默认选择最近的会话。

### 新建会话

用户可以创建新的会话。发送第一条消息时，如果当前没有选中的会话，系统也会自动创建会话。

### 查看会话

用户选择会话后，系统会加载该会话下的历史消息，方便继续之前的学习主题。

### 重命名会话

用户可以修改当前会话标题，用更清晰的名称标记学习主题。

### 删除会话

用户可以删除不再需要的会话。删除当前会话后，前端会清空消息区，并在仍有其他会话时自动切换到下一个会话。

## AI 供应商

系统支持通过后端统一接入不同 AI 供应商：

- OpenAI
- DeepSeek

前端不会直接调用 AI API。所有请求都会发送到 Node.js 后端，由后端根据 `AI_PROVIDER`、模型名称和 API Key 配置选择具体供应商。

这种设计可以：

- 避免 API Key 暴露在浏览器中
- 统一处理鉴权、错误、上下文和消息持久化
- 方便在不同模型供应商之间切换
- 便于后续扩展限流、日志和 token 统计

## 数据持久化

系统使用 MySQL 保存业务数据：

- 用户信息保存在 `users` 表
- 会话信息保存在 `conversations` 表
- 消息记录保存在 `messages` 表

用户重新登录后，可以继续查看自己的历史会话和消息。

## 当前限制

- 当前文档未覆盖完整 API 请求和响应字段，接口细节请查看 `docs/api.md` 和后端路由实现。
- 搜索、归档、模型参数调节等能力尚未形成完整产品说明，后续可以作为扩展功能补充。
- AI 回复质量和速度取决于后端配置的供应商、模型和网络环境。
